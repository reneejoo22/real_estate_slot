# 부동산 매물 검색 챗봇 (자연어 슬롯 필링 기반)

자연어 입력에서 부동산 검색 조건(슬롯)을 추출하고, 조건에 맞는 매물을 찾아 응답하는 챗봇 프로젝트입니다.
LangChain + Gemini function calling으로 슬롯을 추출하고, 매물 검색은 pandas 필터링 + 점수 기반 랭킹으로 처리합니다.

## 파이프라인 구조

```
사용자 발화
   │
   ▼
① 의도 분류 (Text Classification, LLM)
   │  잡담이면 여기서 종료
   ▼
② 슬롯 추출 (Few-shot Prompt Engineering + Information Extraction, LLM Function Calling)
   │  지역 / 거래유형 / 주거유형 / 층조건 / 근접시설 / 가격_최소 / 가격_최대 / 면적_최소 / 기타
   ▼
③ 검색 (일반 코드 + 로컬 임베딩 모델, AI API 호출 없음)
   │  ┌ 필수 조건: 지역/거래유형/주거유형/가격/면적 → pandas 필터링 (정확 일치/범위)
   │  ├ 가산점 조건: 층조건/근접시설 → 일치하면 +1점, 아니어도 탈락 없음
   │  └ 기타 슬롯이 있을 때만: paraphrase-multilingual-MiniLM-L12-v2로 매물 설명과 유사도 계산 → 점수에 반영
   │    ※ 반려동물/주차처럼 가능/불가가 명확한 속성은 True/False 컬럼 기반 하드 신호를 우선 적용
   ▼
④ 결과 포맷팅 — 점수 상위 3개 매물 (템플릿 기반, LLM 호출 없음)
   ▼
최종 답변 출력
```

## 슬롯 스키마

| 슬롯 | 타입 | 설명 |
|---|---|---|
| 지역 | 범주형 (자유 텍스트) | 구/동/역 단위 지역명 |
| 거래유형 | 범주형 | 전세 / 월세 |
| 주거유형 | 범주형 | 원룸 / 투룸 / 오피스텔 |
| 층조건 | 범주형 | 반지하 / 1층 / 저층 / 중층 / 고층 |
| 근접시설 | 범주형 | 역세권 / 대학가 인근 / 학교 근처 / 마트 도보 5분 / 버스정류장 인접 |
| 가격_최소 | 숫자 | 만원 단위 |
| 가격_최대 | 숫자 | 만원 단위 |
| 면적_최소 | 숫자 | 평수 |
| **기타** | **자유 텍스트** | **위 슬롯에 해당하지 않는 자유 조건. 반려동물 가능 여부, 주차, 채광, 분위기, 즉시 입주 등. Few-shot 예시를 통해 핵심 의미를 짧게 정규화해서 추출.** |

## Few-shot 프롬프팅

`기타` 슬롯처럼 자유 표현이 다양한 경우, 규칙 없이 구조화된 출력을 뽑기 어렵습니다. 이를 해결하기 위해 `extract_slots` 함수의 시스템 프롬프트에 실제 사람 발화 스타일의 예시 4개를 포함했습니다.

예시 문장 수준: `"성동구에서 월 50에 우리 강아지랑 살만한 곳"` → `{"지역": "성동구", "거래유형": "월세", "가격_최대": 50, "기타": "반려동물 동반 가능한 곳"}`

Few-shot 설계 원칙:
- 애매하면 억지로 채우지 않고 null 우선
- 정형 슬롯으로 잡을 수 있는 건 기타에 넣지 않음 ("지하철역 5분" → 근접시설: 역세권)
- 기타는 사용자 원문이 아니라 핵심 의미를 정규화한 표현으로 저장 ("강아지랑 살 수 있는" → "반려동물 동반 가능한 곳")

## 기타 슬롯이 채워졌을 때의 검색 흐름

기타 슬롯이 **null이면** 임베딩 모델을 로드하지 않습니다. 기타 슬롯에 값이 있을 때만 아래 과정이 추가됩니다.

1. `paraphrase-multilingual-MiniLM-L12-v2` (sentence-transformers, 다국어 지원) 를 로컬에서 로드
   - 최초 1회만 인터넷에서 다운로드(약 470MB), `models_cache/` 폴더에 저장
   - 이후 실행부터는 `HF_HUB_OFFLINE=1`로 로컬에서만 즉시 로드 (인터넷 불필요)
2. 기타 슬롯 텍스트를 벡터로 변환
3. 필수 조건 필터를 통과한 **후보 매물들의 `설명` 컬럼**과 코사인 유사도 계산
4. 유사도를 기존 점수(층조건/근접시설 가산점)에 더해 최종 랭킹 결정
5. 반려동물/주차처럼 "가능 vs 불가능"이 명확히 갈리는 속성은 임베딩이 두 방향을 헷갈리는 경우가 있어, 매물 데이터의 `반려동물가능`/`주차가능` True/False 컬럼을 하드 신호로 우선 적용(가중치 3배)하고 임베딩 유사도는 보조 지표(0.3배)로만 씀

## 파일별 설명

### 데이터 정의 (`db/`)
- **`slot_schema.json`** — 슬롯 스키마 정의. 각 슬롯의 타입(범주형/비범주형), 허용 값, 설명 포함. `langchain_pipeline.py`의 `SlotFilling` Pydantic 클래스와 1:1 대응.
- **`synthetic_listings.csv`** — 합성 매물 DB. 챗봇이 실제로 검색 대상으로 사용. 반려동물가능/주차가능 컬럼 포함.
- **`test_cases.csv`** — 슬롯 추출 성능 평가용 테스트셋. `utterance`(입력)와 `true_*`(정답 슬롯) 컬럼으로 구성. 기타 슬롯 케이스 포함.

### 데이터 생성 (`make_data/`)
- **`generate_synthetic_data.py`** — 범주형 슬롯 조합(카테시안 곱)으로 매물을 합성 생성. 각 매물마다 반려동물/주차/채광/분위기 등 속성을 랜덤 조합해 1~4문장 자유 텍스트 설명(`설명` 컬럼)을 함께 생성. 결과물: `db/synthetic_listings.csv`
- **`generate_test_utterances.py`** — 매물 데이터에서 슬롯 일부만 골라 자연어 발화를 생성하고, 발화에 실제로 등장한 슬롯만 정답으로 저장. 기타 슬롯 테스트 케이스(반려동물/주차/채광 등 자유 표현)도 포함. 결과물: `db/test_cases.csv`

### 핵심 파이프라인
- **`langchain_pipeline.py`** — 메인 파이프라인. 의도 분류 → Few-shot 기반 슬롯 추출 → 점수 기반 검색(기타 슬롯이 있을 때만 임베딩 모델 호출) → 결과 포맷팅 순서로 처리. LLM은 Gemini(`gemini-2.5-flash`), 임베딩은 로컬 `paraphrase-multilingual-MiniLM-L12-v2` 사용.

### 평가 (`result/`)
- **`evaluate_pipeline.py`** — 테스트셋 전체에 대해 슬롯 추출 → 검색 → 결과 생성을 실행하고 정답과 비교 채점. 정형 슬롯은 정확 일치(Accuracy/F1), 기타 슬롯은 코사인 유사도(임계값 0.5) 기반으로 채점. 평가 완료 후 자동으로 `score_existing_results.py`를 호출해 전체 누적 결과를 재집계. Gemini 무료 티어 한도를 고려해 호출 사이에 `time.sleep`을 두고, `max_samples`로 테스트 건수 제한 가능.
- **`score_existing_results.py`** — LLM 없이 이미 쌓인 `eval_results.csv`만 읽어서 지표를 재계산. API 제한으로 평가가 중간에 끊겼을 때 사용.
- **`eval_results.csv`** — 건별 누적 결과. 입력/예측 슬롯/정답 슬롯/슬롯별 일치 여부/기타 유사도 점수/상위 3개 매물/최종 답변 포함.
- **`eval_summary.csv`** — 슬롯별 Accuracy/Precision/Recall/F1 요약. 전체 일치율(엄격)과 슬롯별 평균 정확도(관대) 두 가지 종합 지표 포함.

### 환경 설정
- **`.env.example`** — API 키 예시. `.env`로 복사 후 값 입력 필요. `.env` 자체는 git에 커밋하지 않음.
- **`models_cache/`** — 로컬 임베딩 모델 저장 폴더. 최초 실행 시 자동 생성. git에는 올라가지 않음(`.gitignore`에 추가됨).

## 실행 순서

```bash
# 패키지 설치 (최초 1회)
pip install langchain-google-genai sentence-transformers python-dotenv pandas scikit-learn

# 1. 매물 데이터 생성 (make_data/ 안에서 실행)
python make_data/generate_synthetic_data.py

# 2. 평가용 테스트 발화 생성
python make_data/generate_test_utterances.py

# 3. 파이프라인 평가 실행 (프로젝트 루트에서 실행)
#    기본값: 10개 샘플, 호출 간 15초 대기, 완료 후 자동으로 score_existing_results.py 호출
python evaluate_pipeline.py

# (선택) API 호출 없이 기존 결과만 재채점
python result/score_existing_results.py
```

## 평가지표 해석

- **ALL_SLOTS_EXACT_MATCH**: 모든 슬롯이 전부 일치해야 정답. 하나만 틀려도 0점이라 낮게 나오는 것이 자연스러움.
- **AVERAGE_PER_SLOT_ACCURACY**: 슬롯별 정확도/매치율의 평균. 실제 성능을 더 잘 반영하는 관대한 지표.
- **기타 슬롯 채점**: 자유 텍스트라 정확 일치 대신 코사인 유사도(임계값 0.5 이상이면 일치로 간주)로 채점. `eval_summary.csv`의 `f1_macro` 컬럼에 평균 유사도 점수가 기록됨.
- 층조건/근접시설은 문장에서 명시적으로 언급되지 않는 경우가 많아 정확도가 다른 슬롯보다 낮게 나올 수 있음.

## 알려진 제약사항

- Gemini API 무료 티어는 분당/일일 요청 횟수 제한이 있어 대량 테스트를 한 번에 돌리기 어려움.
- 부정 표현("반지하 말고" 등)은 현재 슬롯 스키마에서 별도로 처리하지 않음.
- 기타 슬롯의 임베딩 유사도 채점은 Gemini가 few-shot 예시대로 핵심 의미를 정규화한 표현으로 추출했을 때를 전제로 함.
